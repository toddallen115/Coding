Closure Exercise

ex. 1 Bank Login

This exercise has a function that creates a bank account for a user. It takes in a username and password. In order to login to your created account the account has a login command. When you have multiple users it is important to note that the passwords for each user must be kept separate (only available in the local scope of the function), or else users might have access to passwords of their fellow users.

We have created a user Horizons with password horizonites. Currently it's possible to access the privatePassword keyword from anywhere. Your job is to move it to inside the login function and see what happens.

ex. 2 Revisit Once

For this exercise we are going to modify the once function you wrote in the prepwork. You have been given the solution to that exercise and are expected to create a version of once that captures the first value it receives. When it is run multiple times the newly created once function should keep outputting the first output. Examples can be found in the comment/description above the function in closure.js

ex. 3 Counter

Enter two numbers, and the label will display all numbers in between the two (incl. the numbers themselves). Fix the function called "functionFactory" using closures.
